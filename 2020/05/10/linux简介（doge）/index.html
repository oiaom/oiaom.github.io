linux的发展历程

这一部分其实本不需要我讲，大家可以百度一下。

https://baike.baidu.com/item/linux/27050?fr=aladdin

但是百度不说人话，搞什么CPU啊线程啊我反正不理解。在我桌面版本的理解看来，就是快。首先占用内存低，即使在虚拟机上也能运行还比较流畅。

但我还是说一下吧，在linux发展史上有几个让我很震撼的点。第一点，就是它的开源免费。什么是开源免费？开源指开放源代码（鬼知道这是什么），源代码在我看来就是一个系统最核心的部分，只要有了它就可以自己修改系统最核心的东西。例如，windows不是开源，意味着并不是谁都能对它进行修改。由于有了开源，linux在全世界迅速传播，那些计算机大神们纷纷下载，自己或者组成团队来将linux朝着自己喜欢的方向改动。到现在为止，全世界linux的版本数量远大于win和mac的各个版本的和，只要一个人有这个能力，他就可以自己下载一个系统最核心的部分并自己对其进行修改。当然，这也是linux无法大面积推广的原因，linux用户力量强大却过于分散，发行版太多了让那些想给linux适配软件的人看了感到无力，所以linux上的大软件比较少（实际上软件总数是比win要多的，但是主流的比较少，其实你也可以这样想，什么是主流软件，不就是win下的软件吗？因为win是主流操作系统，所以linux下缺少主流软件也可以理解了哈哈，我可真是逻辑鬼才）。当然，这样一个伟大的思想也让linux拥有特别强大的生命力。当然还有其他的优点，主要是服务器版本的。桌面版本只是入门，如果没有使用服务器版本其实就不能算是真正的linux用户！虽然我是使用桌面版本的，但是我隐约感觉服务器版才是linux的精髓，但是我现在没有能力使用，桌面版应该只是一个过渡，等我知识积累到一定程度，我会去尝试服务器版的。

正是由于这一点，来自全世界的无数程序员参与了Linux的修改、编写工作，程序员可以根据自己的兴趣和灵感对其进行改变，这让Linux吸收了无数程序员的精华，不断壮大。

​ —-《百度百科》

第二点就是它的基本思想很有意思：

Linux的基本思想有两点：第一，一切都是文件；第二，每个文件都有确定的用途。其中第一条详细来讲就是系统中的所有都归结为一个文件，包括命令、硬件和软件设备、操作系统、进程等等对于操作系统内核而言，都被视为拥有各自特性或类型的文件。至于说Linux是基于Unix的，很大程度上也是因为这两者的基本思想十分相近。

​ —-《百度百科》

一切皆文件，一切都有确定的用途。当我们在win上运行文件的时候，我们有没有想过为什么这个东西一点开就会运行？它的里面是什么？我们很难查看到，因为你一点它就开始运行了，它的内部是什么就很模糊。（虽说你也可以尝试一下用记事本打开exe文件，反正我当初没有试过）当初我还在用win的时候，总是喜欢把exe看成一个“程序”。而在linux当中，以我的mint为例，若一个安装包的格式是deb包，那么我可以使用安装包程序打开，它就会自动帮我安装。但是如果我不使用安装应用的软件打开呢？例如我将它视作压缩包，用归档管理器打开，WOW！我看到了两个tar.gz格式的安装包！我还可以继续打开，找到里面的一个命令文件，并查看它的命令。写的很清楚，大意就像将安装包里面的哪一个文件移动到哪一个位置，然后创建桌面快捷方式，然后设置触发器呀，将什么指向什么啊之类的。这其实也是一个在Linux上安装软件的重要步骤，对于一般人，我们喜欢一键安装的deb包，但在很多软件下载的时候你选择linux版本它却给了你一个压缩包，有可能是tar.gz的，你可以把这个文件移到这个位置，那个文件移到那个位置………经过多次这样操作以后，软件就安装上了，这就叫手动安装（当然现在一般的情况tar.gz包里面会附带一个可以执行的文件，为了防止手动安装的繁琐，可以直接尝试使用命令运行它，然后再输入一些其他的命令，然后在配置一些东西，这就是linux上的编译安装，比一键安装听上去繁琐，但命令行熟练了以后反而快一些，甚至可以不要安装软件的软件，直接手动编译安装！）这就是一切皆文件的思想，安装的本质就是将安装包里面的文件移到各自合适的位置，然后确保他们之间有联系，例如当我点击应用程序的图标的时候它要能作为引子继续唤醒其他地方的文件，如果不行就会显示路径错误。

那么这两个点是我关注比较深的点，其他的像什么兼容啊什么的，都不是我这个日常用户所关注的了。

Linux与windows使用上的不同点

在详细介绍Linux各个发行版之前，先要从大体上区分Linux和windows，当然我这里主要从日常使用体验上区别，那些高深的等我把标题的doge去掉之后就会讲的。

第一点，就是文件系统的区别，各个主要的文件系统在这篇博客上已经列举出来了，虽说没有逐个介绍，但是也可以百度啊，逐个百度就好，我想这也是作者没有介绍的原因。

Linux#

1、Linux：存在几十个文件系统类型：ext2，ext3，ext4，xfs，brtfs，zfs（man 5 fs可以取得全部文件系统的介绍）

不同文件系统采用不同的方法来管理磁盘空间，各有优劣；文件系统是具体到分区的，所以格式化针对的是分区，分区格式化是指采用指定的文件系统类型对分区空间进行登记、索引并建立相应的管理表格的过程。

  • ext2具有极快的速度和极小的CPU占用率，可用于硬盘和移动存储设备
  • ext3增加日志功能，可回溯追踪
  • ext4日志式文件系统，支持1EB（1024*1024TB），最大单文件16TB，支持连续写入可减少文件碎片。rhel6默认文件系统
  • xfs可以管理500T的硬盘。rhel7默认文件系统
  • brtfs文件系统针对固态盘做优化，
  • zfs更新？

注：EXT（Extended file system）是延伸文件系统、扩展文件系统，ext1于1992年4月发表，是为linux核心所做的第一个文件系统。

格式化命令：mkfs -t <文件系统类型> <分区设备文件名>

　　　　　　mkfs.xfs /dev/sdb1

man 5 fs可以取得全部文件系统的简要介绍

最大支持文件等信息？

windows#

  • FAT16：MS—DOS和win95采用的磁盘分区格式，采用16位的文件分配表，只支持2GB的磁盘分区，最大单文件2GB，且磁盘利用率低
  • FAT32：（即Vfat）采用32位的文件分配表，支持最大分区128GB，最大文件4GB
  • NTFS：支持最大分区2TB，最大文件2TB，安全性和稳定性非常好，不易出现文件碎片。

其他#

  • RAMFS：内存文件系统
  • ISO 9660：光盘
  • NFS：网络文件系统
  • SMBAFS/CIFS：支持Samba协议的网络文件系统

出处是：https://www.cnblogs.com/itplay/p/10991266.html#1201740238

但是这些文件系统的优劣感觉和一般使用者距离很远，所以直接说实用的。在windows下，无法读取以ext4作为文件系统的U盘，在win下查看磁盘管理的时候，只能感觉到那里有一个分区，却无法读取到用了多少更别说内容了（win10和Ubuntu双系统下的测试）。但是在linux下似乎可以将win的磁盘视作U盘而读取其中的消息。你可以对它进行挂载或卸载。NTFS是win的常用格式，ext4是linux的常见格式。其实可以在windows下下载一些能够查看ext4文件系统的软件以实现在windows下访问双系统的linux文件。

第二点，就是内存使用上的区别。虽然mint感觉内存占用少，但是Ubuntu的内存却一直居高不下，其实并不是说内存占用越少计算机就越快，相反，给人感觉ubuntu要更快一些。可以理解成一个简单的动作，ubuntu都用了全力去做，而windows保留了一部分力量。下面这一篇文章我觉得很好：

https://www.imydl.tech/linux/253.html

具体可以自己看链接，就不赘述了。所以内存高不一定慢。（其实我感觉win10也有这个特性，内存占用也比较高但用着也不慢，当然有的人内存占用高就很卡，我不太清楚，应该流畅度还是和CPU有关系）

第三点，软件安装方面的差异。首先声明，linux是无法运行exe文件的。而大多数软件的安装程序是exe的（本来就是开发给win用的233），你点开exe后缀的文件，只会发现一些看不懂的英文。所以不要试图通过exe文件来安装应用。linux的安装包是其他格式的。例如，debian系的是deb后缀的安装包，是因为debian系的包管理器是dpkg。而红帽(Redhat)系的包管理器则是RPM，所以可以安装rpm后缀的安装包。当然还有其他linux系统，这些安装方式就比较深奥了。但其实安装的实质本身就是把文件放在哪一个位置，可不可以执行只是一个方便不方便的问题。即使是包管理器为dpkg的发行版，也可以安装rpm包（前提是包里面的东西支持这个系统，例如我同样的文件，然后一个打包成deb，另一个打包成rpm，那这两个安装包其实是一样的，因为里面的文件是一样的）。rpm或者是deb只是识别的问题，如果是对应的格式和包管理器的话，可以直接命令行或者是系统自带的图形化的安装包管理器自动安装，很方便。如果不对应，那就要手动安装，把文件手动放到指定位置了，当然也可以格式转换工具转换，这在百度上都可以搜到。

这个我使用的比较少，基本就是安装体验了一下。下载镜像的过程中很快，镜像不到一个G。但是如果你需要桌面环境的话就悲剧了，安装过程就会非常慢，当然这不是重点。我试了一下，好像无法安装WPS，可能是缺少什么架构啊之内的，这个不是很清楚。总之这个不适合日常使用。debian10的安装界面还是很友好的，还是一个图形界面，也可以装一个来学习，毕竟是一个老祖宗级别的系统，学透了这个就对所有衍生版有了基础。而且由于很老，时间长，关于依赖问题这一方面解决的很好，相当稳定，同样因为老，在一些支持新软件的问题上面不太好。

Kubuntu Lubuntu Xubuntu

这三个一看就是Ubuntu的亲戚。他们和Ubuntu的不同其实就体现在桌面环境的不同。Ubuntu是gnome桌面，kubuntu是KDE桌面，Lubuntu是LXDE桌面然后Xubuntu是xfce桌面。那么具体日常使用过程中有什么不同呢？

首先是就是易不易于使用和美观的问题了，除了Ubuntu以外，其他桌面的按钮布局和windows比较相似，但是Kubuntu的桌面效果最为漂亮华丽，而且可以自由定制得非常漂亮，自然占用内存也比较高，gnome排第二，xfce和lxde占用差不多。美观易用和占用内存往往是很难平衡的。我3.5G运存，Lubuntu开机占百分之十几。

命令的差异。曾经有人问，这些发行版既然都是基于Ubuntu而仅仅是桌面不一样，那么命令也应该和Ubuntu完全一样才对，但其实还是有差别的。桌面环境可以理解成我们使用者和系统中间的桥梁，我们的点击命令系统听不懂，借助图形环境让系统将点击转化成明确的指令（个人理解O(∩_∩)O哈！）。那么桌面不一样那为什么命令也不一样呢？终端不是应该直接和系统接触吗？其实我们打开的终端并不是真正的终端，我们打开的终端只能叫做终端模拟器，你见过真正的终端能补全命令吗？关于哪里能接触到真正的终端的问题，打开archlinux就能看到真正的终端了。那就是真的全黑，只有一个光标再闪烁，我在虚拟机上无数次尝试之后至今还没有装上archlinux。（后来又研究了一下，发现要是有时间给我折腾我应该能装上吧，但是没时间啊问题是）那么例如，我想以管理员权限打开一个新的文件管理窗口，因为我想删除这个文件但是权限不够。在Ubuntu中，以root用户打开文件管理的命令比较简单，使用时间比较长的人应该就知道了：

sudo nautilus

在Ubuntu中运行此命令就可以以root打开一个文件管理的窗口。然而在mint linux（关于为什么冒出来了一个mint linux是因为我忘了在这三个buntu里面以管理员打开的命令是不是一样了，好像是，又好像不是^_^）

sudo Thunar

为什么呢？因为这样打开的是一个图形界面！图形界面是由桌面提供的！你要用管理员权限在图形界面操作就必须要和桌面商量好，而桌面环境不一样，一个用nautilus作为文件管理器，一个用Thunar作为文件管理器。那如果不需要图形界面来操作，那进入root用户毫无疑问，是通用的：

sudo su

所以，各个版本的命令上的差异是这样产生的，当命令设计到图形的时候，一般就会出现差异，因为桌面环境不同，但是如果不涉及图形，像安装软件包的命令都是apt-get啊（这里指debian一系），还有sudo表示管理员权限啊，这些都是一样的。

还有，似乎这三个发行版完美地延续了WPS打开容易出问题的漏洞，不是打开慢就是根本打不开，我也是很佛啊。不过由于这三个“儿子”有类似windows右下角的状态栏，用他们上QQ是能够享受到小图标闪烁的，但是不知道为什么，QQ老是不能长时间挂着，总是喜欢自动退出。如果电脑内存比较充裕，我还是很推荐Kubuntu的，体验非常好！QQ问题和WPS问题折腾一下我估计还是可以解决的。毕竟用Kubuntu的人比较多，说明大部分问题可以解决。

mint linux

终于到了我现在使用的系统了。这个系统也是基于Ubuntu的，但是前面Ubuntu的三个儿子几乎和Ubuntu同时更新或者差别不大，lubuntu的20.04就是基于Ubuntu的20.04的。而mint是落后Ubuntu一个版本，比如我现在使用的19.3或者19.2、19.1和19.0都是基于Ubuntu18.04做出来的，当然软件库一脉相承了。正因为落后一个版本，所以会修复很多的bug，这也导致mint是公认比Ubuntu好用的发行版，但是就是版本有一点低。那么mint有哪些地方易于使用呢？

在介绍mint易于使用之前，必须了解到，mint实际上下载镜像的时候有三个选择，分别是mate、Cinnamon、和比较轻量的xfce，我现在的版本：mint linux 19.3 xfce

不知道怎么回事，好像只有xfce的版本能够正常的挂长时间QQ，不过偶尔会出现闪退的情况，其他的桌面环境好像和QQ有仇一样，长时间登录就直接闪退，当然这个QQ官方有很大关系。还有就是由于整个mint系列预装的输入法都是fictx，也不会出现像ubuntu一样自带的ibus输入法和WPS不兼容的问题，而且在ubuntu下卸载ibus要非常小心，一不小心整个桌面都卸载掉了，毕竟是和桌面一起的。所以对整个mint来说，使用WPS体验都非常好，mint当中，只有xfce环境使用QQ体验比较好（当然也有可能是偶然，毕竟其他桌面环境我也没有用多长时间，QQ老是闪退）。

非常流畅，纵享丝滑。xfce开机占用内存15，非常离谱，打开什么都比较快。即使是其他桌面环境开机占用内存也不到30，非常适合我这种老旧的台式机使用。当然啦，因为占用内存小，可以开虚拟机，能够分配很大一部分出来体验其他的系统，如果是在Ubuntu上开虚拟机，是非常卡顿的体验，所以由于内存小可以在上面体验其他的发行版。

其实优点也差不多说完了，优点之中最重要的当然就是流畅。毕竟mint是Ubuntu优化过后的产物，所以同版本的mint和ubuntu肯定是mint好用（这里的同版本是指基于哪一个优化的，像mint19.3和Ubuntu18.04是“同版本”），但是由于mint总是比Ubuntu要慢一个版本，所以应该要将mint19.3和Ubuntu20.04来进行对比比较好。以下是我在使用过程中的对比结果：

Ubuntu20.04最突出的地方就是软件源比较新，而且安装wine比较简单。有了wine，ubuntu就可以运行一些在windows下才能运行的程序，但是操作肯定还是有些困难的，需要对wine进行一些配置（这详细入坑又是一篇比这还长的文章！）我使用Ubuntu的时候就会在钉钉上听网课。人家ubuntu下一条：

sudo apt-get install wine64

就能搞定的问题不知道为什么在mint下就要折腾，一会缺少这个依赖，一会缺少那一个依赖。所以Ubuntu的镜像大，那么它附带的东西就比较全。
